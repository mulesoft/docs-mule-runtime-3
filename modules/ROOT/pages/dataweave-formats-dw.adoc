[[properties]]
== Configuration Properties

DataWeave supports the following configuration properties for the Dw format.


[[reader_properties]]
=== Reader Properties

The Dw format accepts properties that provide instructions for reading input data.


[cols="1,1,1,3a", options="header"]
|===
|Parameter |Type |Default|Description
|`externalResources`  |`Boolean`|`false`|Allow to read external entities on readUrl. Valid Options: `true` or `false`
|`javaModule`  |`Boolean`|`false`|Allow to load java module functions. Valid Options: `true` or `false`
|`onlyData`  |`Boolean`|`false`|Marks that this DataWeave will have only data. Valid Options: `true` or `false`
|`privileges`  |`String`|`''`|Allowed privileges to be use in the format. It is a comma separated value i.e 'Resources,Properties'
|===

[[writer_properties]]
=== Writer Properties

The Dw format accepts properties that provide instructions for writing output data.

[cols="1,1,1,3a", options="header"]
|===
|Parameter |Type |Default|Description
|`bufferSize`  |`Number`|`8192`|The size of the buffer writer
|`deferred`  |`Boolean`|`false`|Indicates if the output should be deferred. Valid Options: `true` or `false`
|`ignoreSchema`  |`Boolean`|`false`|If the writer should ignore or not the schema. Valid Options: `true` or `false`
|`indent`  |`String`|`'  '`|The string that is going to be used as indent.
|`maxCollectionSize`  |`Number`|`-1`|The maximum amount of elements that are allowed in an Array or an Object if -1 then no limitation is set.
|`onlyData`  |`Boolean`|`true`|Marks that this DataWeave will have only data. Valid Options: `true` or `false`
|===
